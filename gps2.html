About the game

In the lawless heart of the Wild West, you are a gunslinger with one goal to be the fastest, deadliest shot in town. To prove your worth, you seek out the four greatest gunslingers. But before your first duel, a mysterious cowboy appears, offering you a golden gun—an unrivaled weapon that guarantees victory with every headshot.

The enemy system features a flexible, phase-based wave spawner that allows designers to easily create multi-stage combat encounters. Each phase supports different enemy types, spawn points, patrol paths, delays, and special behaviors (like hostage spawns or checkpoints).

Phase-based enemy spawning with staggered delays and patrol routes

EnemySpawner.cs — Phase System

EnemySpawner.csC#
public class EnemySpawner : MonoBehaviour 
{
    [System.Serializable]
    public class Phase 
    {
        public string phaseName;
        public Transform[] spawnPoints;
        public Transform[] walkPoints;
        public bool[] spawnNormalEnemy;
        public bool[] spawnStrongEnemy;
        public bool[] spawnHostage;
        public float[] spawnDelays;
        public bool isCheckPoint;
    }

    private int totalEnemiesInCurrentPhase = 0;
    private int enemiesKilled = 0;

    IEnumerator SpawnEnemiesInPhase(Phase phase)
    {
        for (int i = 0; i < phase.spawnPoints.Length; i++)
        {
            yield return new WaitForSeconds(phase.spawnDelays[i]);

            if (phase.spawnNormalEnemy[i]) 
                SpawnEnemy(normalEnemyPrefab, phase.spawnPoints[i]);

            if (phase.spawnStrongEnemy[i]) 
                SpawnEnemy(strongEnemyPrefab, phase.spawnPoints[i]);

            totalEnemiesInCurrentPhase++;
        }
    }

    public void RemoveEnemy()
    {
        enemiesKilled++;
        if (enemiesKilled >= totalEnemiesInCurrentPhase)
        {
            EndPhase();   // Move to next phase or trigger checkpoint
        }
    }
}

What this does

The EnemySpawner uses a serialized Phase class that designers can configure directly in the Inspector. Each phase supports staggered spawning with individual delays, different enemy types, patrol routes, and optional checkpoint behavior.

When enemies die, they call RemoveEnemy(), which tracks progress and automatically advances to the next phase once all enemies in the current wave are defeated.

Key Advantages

• Fully designer-friendly — no coding needed to set up waves
• Supports multiple enemy types per phase (normal, strong, hostage)
• Staggered spawn delays create more dynamic combat pacing
• Integrated with patrol system using walkPoints
• Checkpoint support to trigger events after clearing a phase
• Clean separation between spawning logic and individual enemy AI

The Audio Manager is a lightweight, reusable system for playing sound effects in the game. It supports random sound variation, dynamic spawning at specific locations, and automatic cleanup — making it easy for designers and other scripts to trigger high-quality audio feedback without performance issues.

AudioManager.cs — Core Sound System

AudioManager.csC#
public class AudioManager : MonoBehaviour 
{
    [SerializeField] private AudioSource soundFXObject;

    public void PlayRandomSoundFxClip(AudioClip[] audioClip, Transform spawnTransform, float volume)
    {
        // Pick a random sound from the array
        int rand = Random.Range(0, audioClip.Length);

        // Create audio source at the position
        AudioSource audioSource = Instantiate(soundFXObject, spawnTransform.position, Quaternion.identity);

        // Play the sound
        audioSource.clip = audioClip[rand];
        audioSource.volume = volume;
        audioSource.Play();

        // Auto-cleanup when done
        float clipLength = audioSource.clip.length;
        Destroy(audioSource.gameObject, clipLength);
    }
}

What this does

This method allows any script to easily play a random sound effect from an array at a specific world position. It instantiates a temporary AudioSource, plays the chosen clip, and automatically destroys the object after the sound finishes — preventing memory leaks and keeping the scene clean.

Key Advantages

• Supports **random sound variation** — prevents repetitive audio fatigue
• Spawns sound at exact location for proper 3D spatial audio
• Fully configurable volume per call
• Automatic cleanup using `Destroy` with delay
• Very easy to use from any script (gunshots, footsteps, UI, impacts, etc.)
• Designer-friendly — just assign an array of AudioClips in the Inspector

Usage Example

From any other script (e.g. Gun, Player, Enemy):

audioManager.PlayRandomSoundFxClip(gunshotClips, firePoint, 0.9f);

The Boss Target Spawner is a flexible system that manages multiple phases of boss fights by spawning and cycling through target points. Designers can easily configure different phases with varying target locations and special behaviors like rapid-fire attack zones.

Dynamic target spawning across multiple boss fight phases with rapid-fire zones

BossTargetSpawner.cs — Core Logic

BossTargetSpawner.csC#
public class BossTargetSpawner : MonoBehaviour 
{
    [System.Serializable]
    public class TargetPoint 
    {
        public Transform point;
        public bool isSpam;   // Rapid-fire attacks
    }

    [System.Serializable]
    public class Phase 
    {
        public string phaseName;
        public List targetPoints;
    }

    private List targetPoints = new List();
    private int currentPhaseIndex = 0;

    public void LoadPhaseTargets(int phaseIndex)
    {
        targetPoints.Clear();
        foreach (TargetPoint target in phases[phaseIndex].targetPoints)
        {
            if (target.point != null)
                targetPoints.Add(target.point);
        }
        currentPhaseIndex = phaseIndex;
    }

    public void LoadNextPhaseTargets()
    {
        int nextPhaseIndex = currentPhaseIndex + 1;
        if (nextPhaseIndex < phases.Count)
            LoadPhaseTargets(nextPhaseIndex);
    }
}

What this does

The system uses a serialized Phase class that designers can configure in the Inspector. Each phase contains a list of target points, some marked as isSpam for rapid-fire attack sections.

LoadPhaseTargets() clears the current list and loads all targets for the new phase. LoadNextPhaseTargets() automatically advances to the next phase when called (usually after the player clears the current one).

Key Advantages

• Fully designer-friendly — phases and targets are configured entirely in the Unity Inspector
• Supports multiple phases per boss fight
• Special isSpam flag allows certain targets to trigger rapid-fire attacks
• Clean and reusable — easily extendable for more complex boss behaviors
• Automatic progression with LoadNextPhaseTargets()
