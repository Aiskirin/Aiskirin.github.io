About the game

Petronas Pursuit is a 2D climber where you play as a criminal escaping the police force in a futuristic Kuala Lumpur. You need to climb all the way to the top of the Petronas Twin Towers to escape on your helicopter. But it's not so simple... The police force is on your trail, using drones and flying cars to stop you from escaping. Even the Petronas Twin Towers have engaged their laser defense system! You only have a limited amount of time before you get caught. The clock is ticking...

When the player performs a dash, a dust cloud particle spawns at their feet for visual feedback, while the character moves smoothly forward using DOTween with a natural easing curve. This makes the dash feel responsive, powerful, and satisfying.

Player dashes → Dust cloud spawns at feet + smooth movement

DashEffectSystem.cs

DashEffectSystem.cs C# 
using DG.Tweening;
using UnityEngine;
using System.Collections;

public class DashEffectSystem : MonoBehaviour
{
    [Header("Dash Effect Settings")]
    [SerializeField] GameObject dustCloud;
    [SerializeField] private Vector3 dustOffset = new Vector3(0f, -0.5f, 0f);

    public float dashDistance = 2f;
    public float dashDuration = 0.2f;

    private bool isDashing = false;

    public IEnumerator DoDash(Vector2 dir)
    {
        isDashing = true;

        // Spawn dust particle effect at player's feet
        if (dustCloud != null)
        {
            Instantiate(dustCloud, transform.position + dustOffset, dustCloud.transform.rotation);
        }

        Vector2 startPos = transform.position;
        Vector2 endPoint = startPos + (dir * dashDistance);

        // Smooth dash movement with nice easing
        yield return transform.DOMove(endPoint, dashDuration)
            .SetEase(Ease.OutCubic)
            .WaitForCompletion();

        isDashing = false;
    }
}

What this does

This system handles the core dash mechanic with both visual polish and smooth gameplay feel:

  • Visual Feedback: Instantly spawns a dust cloud prefab at the player's feet (offset slightly downward) to show ground interaction and power.
  • Smooth Movement: Uses DOTween to move the player over dashDuration seconds with Ease.OutCubic. This creates a fast start and natural slowdown, making the dash feel weighty and satisfying.
  • Coroutine: The dash runs as a coroutine so it doesn't block other player systems like animation or input.

How to use

From your Player Controller, call it like this:

if (Input.GetKeyDown(KeyCode.Space) && !dashSystem.isDashing)
{
    Vector2 dashDirection = GetInputDirection();   // or facing direction
    StartCoroutine(dashSystem.DoDash(dashDirection));
}

Tip: Make sure your dustCloud prefab has a Particle System with "Play On Awake" and a short lifetime for best results.

The Drone is a dynamic flying enemy that moves horizontally into the play area, hovers while firing at the player with imperfect aiming, and leaves after firing a set number of shots. It uses arrival steering for smooth deceleration and spawns warning indicators before each shot.

(Recommended: 16:9 showing movement + shooting)
Drone flies in → aims with warning light → fires projectiles → leaves after 3 shots

Drone.cs

Drone.cs C# 
using MoreMountains.Feedbacks;
using UnityEngine;

public class Drone : MonoBehaviour
{
    [Header("Movement")]
    [SerializeField] private float moveSpeed = 3f;
    [SerializeField] private float slowingRadius = 3f;
    [SerializeField] private float fixedX = -6f;
    [SerializeField] private float yOffset = 1f;

    [Header("Shooting")]
    [SerializeField] private GameObject projectilePrefab;
    [SerializeField] private GameObject targetLightPrefab;
    [SerializeField] private GameObject explosionPrefab;
    [SerializeField] private float shootDelay = 1f;
    [SerializeField] private int shotsBeforeLeaving = 3;
    [SerializeField] private float missRadius = 1.5f;

    private bool movingIn = false;
    private bool leaving = false;
    private int shotsFired = 0;
    private Vector3 moveInWorldPos;

    void Update()
    {
        if (movingIn)
        {
            moveInWorldPos = new Vector3(fixedX, player.position.y + yOffset, transform.position.z);
            MoveIn();
        }
        else if (!leaving && player != null)
        {
            shootTimer += Time.deltaTime;
            if (shootTimer >= shootDelay)
            {
                Shoot();
                shootTimer = 0f;
            }
        }
    }

    void MoveIn()
    {
        Vector3 displacement = moveInWorldPos - transform.position;
        float distance = displacement.magnitude;

        // Arrival steering behavior for smooth deceleration
        float speed = distance <= slowingRadius 
            ? moveSpeed * (distance / slowingRadius) 
            : moveSpeed;

        transform.position += displacement.normalized * speed * Time.deltaTime;

        if (distance <= 0.1f) movingIn = false;
    }

    void Shoot()
    {
        animator?.SetTrigger("IsShooting");

        // Calculate spawn position with muzzle offset
        Vector2 spawnPos = (Vector2)transform.position + 
            ((Vector2)player.position - (Vector2)transform.position).normalized * muzzleOffset;

        // Add randomized targeting for realistic imperfect aim
        Vector2 targetPos = (Vector2)player.position + Random.insideUnitCircle * missRadius;

        // Spawn visual warning indicator at target location
        if (targetLightPrefab != null)
            Instantiate(targetLightPrefab, targetPos, Quaternion.identity);

        // Calculate rotation and spawn projectile
        Vector2 dir = (targetPos - spawnPos).normalized;
        float angle = Mathf.Atan2(dir.y, dir.x) * Mathf.Rad2Deg - 90f;

        GameObject proj = Instantiate(projectilePrefab, spawnPos, Quaternion.Euler(0f, 0f, angle));
        proj.GetComponent<Projectiles>()?.SetTarget(targetPos);
        proj.GetComponent<Projectiles>()?.SetExplosionPrefab(explosionPrefab);
        proj.GetComponent<Projectiles>()?.Launch();

        if (++shotsFired >= shotsBeforeLeaving)
            leaving = true;
    }
}

What this does

This Drone enemy creates engaging vertical climbing pressure:

  • Smart Movement: Uses an arrival steering behavior (slows down as it approaches its target X position) for natural, non-abrupt flying motion.
  • Imperfect Aiming: Doesn't always shoot perfectly — adds randomness within missRadius so the player has a fair chance to dodge.
  • Warning System: Spawns a target light prefab at the intended impact point before firing, giving the player a visual cue to react.
  • Limited Engagement: Fires only a set number of shots (shotsBeforeLeaving) then leaves the screen, preventing the area from becoming too overwhelming.

Key Features

• Fixed horizontal entry position with dynamic Y tracking
• Uses MoreMountains.Feedbacks for shooting animation trigger
• Projectile gets rotation based on aim direction
• Easy to tweak via Inspector (speed, shot count, miss amount, etc.)

The Police Car is a horizontally moving obstacle that patrols back and forth using a smooth sine wave. Each car has a randomized phase offset so they don't all move in sync, creating a more natural and challenging dodging experience while climbing.

Police cars moving smoothly left and right using sine wave motion

PoliceCar.cs

PoliceCar.cs C# 
using UnityEngine;

public class PoliceCar : MonoBehaviour
{
    [SerializeField] private float distance = 5f;
    [SerializeField] private float speed = 2f;

    private Vector3 startPos;
    private float phaseOffset;

    void Start()
    {
        startPos = transform.position;
        // Random offset so cars aren't synchronized
        phaseOffset = Random.Range(0f, Mathf.PI * 2);
    }

    void Update()
    {
        // Smooth side-to-side movement using sine wave
        float offset = Mathf.Sin(Time.time * speed + phaseOffset) * distance;
        transform.position = new Vector3(startPos.x + offset, startPos.y, startPos.z);
    }
}

What this does

This simple but effective script creates dynamic horizontal obstacles perfect for a vertical climber:

  • Sine Wave Movement: Uses Mathf.Sin() to create smooth, natural back-and-forth motion.
  • Phase Offset: Each car gets a random starting phase so they move out of sync — preventing predictable patterns and increasing challenge.
  • Configurable: Easy to tweak distance (how wide they patrol) and speed (how fast they move) directly in the Inspector.
  • Lightweight: Very performant and works well even with many cars on screen.

How to use

Simply attach this script to your Police Car prefab. You can:

  • Adjust distance to control how far left/right the car moves
  • Change speed to make cars faster or slower
  • Duplicate the prefab multiple times at different heights for layered obstacles

Tip: Combine with colliders set as triggers or solid obstacles depending on whether the player should die on contact or get knocked back.
