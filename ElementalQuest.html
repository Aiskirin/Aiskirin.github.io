About the game

Elemental Quest is a 3D action-adventure game where you play as Nerlim, a young mage mastering the elements of Fire, Grass, and Water. Princess Lyra has been captured you must journey through the ruined castle of Lord Eclipse, a fallen sorcerer who once tried to fuse Light and Dark magic. His castle gate is locked, needing the souls of 3 mighty heroes that have fallen into his castle grounds while trying to fight Lord Eclipse during a war. Explore overgrown courtyards, flooded docks, and molten forges, using elemental powers to defeat monsters and solve environmental puzzles.

Elemental Quest is currently in development.

Enemies use Unreal's StateTree system combined with the Gameplay Ability System (GAS) to manage states like patrol, chase, attack, and death. Each enemy type has unique ability sets that are activated based on game conditions.

    {

What this does

Uses the Gameplay Ability System to activate enemy abilities by tag. This keeps the AI logic clean and data-driven — each enemy Blueprint just sets which tags map to which abilities, without needing code changes.

A central Puzzle Manager tracks all puzzles in the level. Each puzzle registers itself on begin play and notifies the manager when completed. The manager then checks win conditions and triggers the next game event.

Puzzle manager tracking completion
PuzzleManager.cppC++
void APuzzleManager::OnPuzzleCompleted(APuzzleBase* Puzzle)
{
    CompletedPuzzles.Add(Puzzle);
                                    if (CompletedPuzzles.Num() >= RequiredPuzzles)
    {
                                    TriggerLevelEvent();
    }
}

What this does

Each time a puzzle is solved it calls this function. Once the number of completed puzzles reaches the required count, TriggerLevelEvent() fires — opening a gate, spawning a boss, or playing a cutscene depending on the level.

The Trigger Manager handles all overlap-based events in the level — zone entry, item pickups, enemy aggro triggers. Designers can configure triggers entirely in Blueprint without touching C++ code.

What this does

When the player walks into a trigger zone this fires and broadcasts a delegate. Any Blueprint in the level can bind to OnPlayerEntered and react — no hardcoded references needed.

When the player enters a checkpoint zone, their transform and current game state are saved to the Game Instance. On death, the player is respawned at the last activated checkpoint.

void ACheckpoint::ActivateCheckpoint(APlayerCharacter* Player)
{
                                    if (!bActivated)
    {
        bActivated = true;

What this does

Saves the player's current position to the Game Instance the first time they touch the checkpoint. The bActivated flag prevents it from re-saving if the player revisits. On death, the Game Instance provides the saved transform for respawn.

The player must use their Grass element ability to grow specific bushes in the correct order. Each bush listens for a GAS gameplay event and notifies the Puzzle Manager when activated in the right sequence.

                                    else

What this does

When the Grass ability hits a bush, it checks with the Puzzle Manager whether this bush is next in the expected sequence. Correct order advances progress; wrong order resets it — enforcing the puzzle challenge.

A gate that requires all connected pressure plates to be activated simultaneously. Plates can be held down by pushing objects onto them. The gate C++ class listens to all plate delegates and opens only when all are active at the same time.

gate-puzzle.gif
Gate opening when all plates active
GatePuzzle.cppC++
void AGate::CheckAllPlates()
{
                                    bool bAllActive = true;
                                    for (APressurePlate* Plate : LinkedPlates)
    {
                                    if (!Plate->bIsActive)
            bAllActive = false;
    }
                                    if (bAllActive) OpenGate();
                                    else CloseGate();
}

What this does

Every time any plate changes state it calls this. The loop checks every linked plate — if even one is inactive the gate stays shut. Only when all are simultaneously active does the gate open.

The player must hit all targets within a time limit. A countdown timer is managed in C++ and exposed to Blueprint for UI display. Missing the deadline resets the puzzle.

void ATimePuzzle::Tick(float DeltaTime)
{
                                    Super::Tick(DeltaTime);
                                    if (bActive)
    {
        TimeRemaining -= DeltaTime;
                                    if (TimeRemaining <= 0.f)
                                    ResetPuzzle();
    }
}

What this does

Every frame while active, the timer counts down. When it hits zero the puzzle resets. The TimeRemaining float is BlueprintReadOnly so the HUD widget can display it without needing a C++ reference.

Cutscenes are driven by Unreal's Level Sequencer. A C++ wrapper handles disabling player input, playing the sequence, and restoring control when it ends — keeping all cutscene logic in one place.

cutscene.gif
Intro cutscene playing
CutsceneManager.cppC++
void ACutsceneManager::PlayCutscene(ULevelSequence* Sequence)
{
                                    DisablePlayerInput();
    SequencePlayer = ULevelSequencePlayer::CreateLevelSequencePlayer(
                                    GetWorld(), Sequence, Settings, SequenceActor
    );
    SequencePlayer->OnFinished.AddDynamic(
                                    this, &ACutsceneManager::OnCutsceneFinished
    );
    SequencePlayer->Play();
}

What this does

Locks player input, creates a Level Sequence player, and binds OnCutsceneFinished to the sequence's end delegate. When the cutscene ends, input is automatically restored — no manual timers or Blueprint event chains needed.
