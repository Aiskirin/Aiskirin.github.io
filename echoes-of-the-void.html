About the game

Echoes of the Void is a sci-fi exploration game set in a procedurally generated universe. Players pilot a ship through uncharted star systems, uncovering ancient ruins and piecing together a mystery spanning thousands of years.

The game focuses on atmosphere and discovery over combat. Every playthrough generates a different set of star systems, ensuring no two players share the same journey.

Mechanics

Procedural world generation

Each run seeds a unique universe — star positions, planet types, ruin placements, and enemy patrol routes are all generated at runtime. No two playthroughs are the same.

Ship movement & physics

The ship uses Unreal's Chaos physics with custom drag and thrust curves to simulate zero-gravity momentum. Holding the thruster builds speed; releasing lets inertia carry you forward.

Ruin discovery & lore system

Ancient ruins scattered across planets hold fragments of lore. Players scan ruins with a proximity sensor — the closer you are, the more detail is revealed. Collected fragments assemble into a timeline in the journal.

What I did

  • Built the procedural world generation system in C++ — each star system, planet layout, and ruin placement is seeded and unique per playthrough.
  • Designed ship movement and physics using Unreal's Chaos system, tuned for that weightless space feel with realistic inertia.
  • Created all shaders in HLSL — including the nebula background, engine trails, and holographic UI elements.
  • Implemented a save/load system using Unreal's serialisation pipeline, allowing players to resume at any discovered waypoint.

Code deep dive

ProceduralWorldGen.cpp C++ 
// Seed the random stream with the game session ID
void AWorldGenerator::GenerateUniverse(int32 Seed)
{
    FRandomStream Stream(Seed);

    for (int32 i = 0; i < StarCount; i++)
    {
        FVector Location = FVector(
            Stream.FRandRange(-50000.f, 50000.f),
            Stream.FRandRange(-50000.f, 50000.f),
            Stream.FRandRange(-10000.f, 10000.f)
        );

        AStarSystem* Star = GetWorld()->SpawnActor<AStarSystem>(
            StarClass, Location, FRotator::ZeroRotator
        );

        Star->InitialiseStar(Stream.RandRange(1, 5));
    }
}

What this does

This function generates the entire game universe from a single integer — the Seed. By passing the same seed to FRandomStream, every star position and planet will always be identical for that seed, which is how sharable universe codes work.

The loop runs StarCount times (set in the editor), each time picking a random 3D position within the defined space boundaries. It then spawns a AStarSystem actor at that location and calls InitialiseStar() with a random size value (1–5) to vary the visual scale and number of orbiting planets.

ShipMovementComponent.cpp C++ 
// Apply thrust and drag each physics tick
void UShipMovementComponent::TickComponent(
    float DeltaTime,
    ELevelTick TickType,
    FActorComponentTickFunction* ThisTickFunction)
{
    Super::TickComponent(DeltaTime, TickType, ThisTickFunction);

    if (bThrusting)
    {
        Velocity += GetOwner()->GetActorForwardVector()
                    * ThrustForce * DeltaTime;
    }

    // Apply drag — slows the ship gradually in space
    Velocity *= FMath::Clamp(1.f - (DragCoefficient * DeltaTime),
                             0.f, 1.f);

    // Clamp to max speed
    if (Velocity.Size() > MaxSpeed)
        Velocity = Velocity.GetSafeNormal() * MaxSpeed;

    GetOwner()->AddActorWorldOffset(Velocity, true);
}

What this does

This runs every physics frame to move the ship. When the player holds the thrust key, bThrusting is true and velocity builds up in the direction the ship is facing — the longer you hold it, the faster you go.

The drag line is the key to making space feel right. Instead of stopping instantly, multiplying velocity by a value slightly below 1 each frame means the ship slows down smoothly over time, like coasting. DragCoefficient is exposed to the editor so it can be tweaked without recompiling.

The final clamp ensures the ship never exceeds MaxSpeed regardless of how long thrust is held, preventing runaway velocity bugs.

RuinScanner.cpp C++ 
// Called when player activates the scanner
void ARuinScanner::ScanNearbyRuins()
{
    TArray<AActor*> FoundActors;
    UGameplayStatics::GetAllActorsOfClass(
        GetWorld(), ARuinFragment::StaticClass(), FoundActors
    );

    for (AActor* Actor : FoundActors)
    {
        float Distance = FVector::Dist(
            GetOwner()->GetActorLocation(),
            Actor->GetActorLocation()
        );

        if (Distance <= ScanRadius)
        {
            ARuinFragment* Ruin = Cast<ARuinFragment>(Actor);
            float Detail = 1.f - (Distance / ScanRadius);
            Ruin->RevealLore(Detail);
        }
    }
}

What this does

This scans all ruin fragments in the world and reveals lore based on how close the player is. GetAllActorsOfClass finds every ARuinFragment in the level, then a distance check filters only ones within ScanRadius.

The clever part is the Detail calculation — dividing distance by the scan radius gives a 0 to 1 float. At maximum distance it's near 0 (barely revealed), and right next to the ruin it approaches 1 (fully revealed). This value is passed into RevealLore() which uses it to control how much text and visual detail is shown, rewarding players for physically getting close.
